## âš™ï¸ Setup and Installation

Follow these steps to get a local copy of DataSphere-CRUD-API-Flask-Web-App up and running for development and testing.

### Prerequisites

Ensure you have the following installed:

*   **Python 3.10+**: Download from [python.org](https://www.python.org/downloads/)
*   **uv**: A fast Python package installer and resolver.
    bash
    pip install uv
    

### Installation Steps

1.  **Clone the repository:**
    bash
    git clone https://github.com/chirag127/DataSphere-CRUD-API-Flask-Web-App.git
    cd DataSphere-CRUD-API-Flask-Web-App
    

2.  **Create and activate a virtual environment using `uv`:**
    bash
    uv venv
    source .venv/bin/activate  # On macOS/Linux
    # .venv\Scripts\activate   # On Windows
    

3.  **Install dependencies:**
    bash
    uv sync
    

4.  **Set up environment variables:**
    Create a `.env` file in the root directory and add necessary environment variables (e.g., `DATABASE_URL`, `FLASK_APP`).
    ini
    # .env example
    FLASK_APP=src/app.py
    FLASK_DEBUG=1
    DATABASE_URL=sqlite:///./database.db
    SECRET_KEY='your_secret_key_here'
    

5.  **Initialize the database (if using SQLAlchemy with migrations, e.g., Alembic):**
    *Note: This project provides a basic SQLAlchemy setup. For production, consider integrating Alembic for migrations.*
    bash
    # Example for creating tables based on models.py
    uv run python -c "from src.database.models import db; from src.app import app; with app.app_context(): db.create_all()"
    

### Running the Application

To start the Flask development server:

bash
uv run flask run


The API will typically be available at `http://127.0.0.1:5000`.

## ğŸ“œ API Endpoints

The API provides standard CRUD operations for a resource (e.g., `items`).

| Method | Endpoint             | Description                       | Request Body (Example)                 | Response (Example)                   |
| :----- | :------------------- | :-------------------------------- | :------------------------------------- | :----------------------------------- |
| `POST` | `/api/v1/items`      | Create a new item                 | `{"name": "New Item", "value": 100}`   | `{"id": 1, "name": "New Item", ...}` |
| `GET`  | `/api/v1/items`      | Retrieve all items                | (None)                                 | `[{"id": 1, "name": "Item A", ...}]` |
| `GET`  | `/api/v1/items/<id>` | Retrieve a single item by ID      | (None)                                 | `{"id": 1, "name": "Item A", ...}`   |
| `PUT`  | `/api/v1/items/<id>` | Update an item by ID              | `{"name": "Updated Item"}`             | `{"id": 1, "name": "Updated Item",..}` |
| `DELETE`|`/api/v1/items/<id>` | Delete an item by ID              | (None)                                 | `{"message": "Item deleted"}`        |

*Detailed API documentation (e.g., Swagger/OpenAPI) is highly recommended for production systems.*
